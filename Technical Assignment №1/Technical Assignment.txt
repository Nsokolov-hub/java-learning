Техническое задание (ТЗ) для разработки программы "Калькулятор чаевых"
Цель проекта
Разработать консольное Java-приложение, которое рассчитывает распределение общей суммы счёта с учётом чаевых между несколькими участниками обеда.

Функциональные требования
1. Ввод данных:
Сумма счёта (billAmount):
Тип данных: double
Значение по умолчанию: 1000
Описание: Общая сумма, которую нужно оплатить за обед.
Процент чаевых (tipPercentage):
Тип данных: double
Значение по умолчанию: 10
Описание: Процент от суммы счёта, который будет добавлен в качестве чаевых.
Количество участников (numberOfPeople):
Тип данных: int
Значение по умолчанию: 4
Описание: Количество людей, участвующих в оплате счёта.
2. Расчёты:
Сумма чаевых (tipAmount):
Формула: tipAmount = billAmount * tipPercentage / 100
Описание: Сумма, добавляемая к счёту в виде чаевых.
Общая сумма (totalAmount):
Формула: totalAmount = billAmount + tipAmount
Описание: Итоговая сумма, которую необходимо оплатить.
Сумма на одного человека (perPersonAmount):
Формула: perPersonAmount = totalAmount / numberOfPeople
Описание: Сумма, которую должен заплатить каждый участник.
3. Вывод результатов:
Вывести на экран:

"Сумма счёта" с двумя знаками после запятой.
"Процент чаевых" с двумя знаками после запятой и знаком процента.
"Сколько человек участвовало в обеде" в виде целого числа.
"Общий счёт" с двумя знаками после запятой.
"Процент чаевых" с указанием процента.
"С каждого" с двумя знаками после запятой.
Нефункциональные требования
Программа должна быть написана на языке Java и компилироваться без ошибок.
Использовать класс System.out.printf() для форматированного вывода данных.
Код должен быть структурирован и понятен для чтения, с комментариями, объясняющими основные шаги программы.
Требования к версии Java
Версия Java должна быть не ниже Java 8.

Пример работы
Сумма счёта: 1000,00
Процент чаевых: 10,00
Сколько человек участвовало в обеде: 4
Общий счёт: 1100,00
Процент чаевых: 10.0%
С каждого: 275,00

                  
Критерии приёмки
Программа должна корректно выполнять все расчёты и выводить результаты в соответствии с заданными формулами и форматами.
Все значения должны быть выведены на экран в читаемом формате с соответствующими пояснениями.
Программа должна быть протестирована на различных входных данных для проверки корректности расчётов.
